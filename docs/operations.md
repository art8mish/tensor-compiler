# Операций в нейронных сетях: Conv, Gemm, MatMul, Mul, Add, ReLU

Данный документ содержит подробное техническое описание шести фундаментальных операций, используемых в нейронных сетях: свёртка (Сonv), обобщённое матричное умножение (Gemm), матричное умножение (MatMul), поэлементное умножение (Mul), поэлементное сложение (Add) и функция активации ReLU. Для каждой операции приведены формальные определения, размерности входных и выходных тензоров, математические формулы и описание алгоритма выполнения.

## 1. Conv

Свёртка — основная операция в свёрточных нейронных сетях (CNN). Она применяется к многоканальным входным данным (например, изображениям) с использованием набора обучаемых фильтров (ядер). Результатом является карта признаков (feature map), которая затем может передаваться дальше по сети.

Ниже приведено детальное описание всех компонентов операции двумерной свёртки (2D convolution), включая размерности, параметры и формулы вычислений. Для определённости будем придерживаться формата данных, принятого в PyTorch: **NCHW** (batch size, channels, height, width). Однако общие принципы легко переносятся на другие форматы (например, NHWC).

### 1.1. Входные данные X (Input tensor)

Входной тензор $ X $ представляет собой набор данных, подаваемых на свёрточный слой. Его размерность:

```math
X \in \mathbb{R}^{N \times C_{\text{in}} \times H_{\text{in}} \times W_{\text{in}}}
```

где:
- $ N $ — размер мини-батча (количество независимых примеров);
- $ C_{\text{in}} $ — количество входных каналов (например, для RGB-изображения $ C_{\text{in}} = 3 $);
- $ H_{\text{in}} $ — высота (пространственное измерение) каждого входного тензора;
- $ W_{\text{in}} $ — ширина каждого входного тензора.

### 1.2. Ядро свёртки W (Weights / Filters)

Ядро свёртки — это набор обучаемых фильтров. Каждый фильтр отвечает за выделение определённого признака. Размерность:

```math
W \in \mathbb{R}^{C_{\text{out}} \times C_{\text{in}} \times kH \times kW}
```

где:
- $ C_{\text{out}} $ — количество выходных каналов (количество фильтров);
- $ C_{\text{in}} $ — количество входных каналов (должно совпадать с $ C_{\text{in}} $ из X);
- $ kH $ — высота ядра (kernel height);
- $ kW $ — ширина ядра (kernel width).

Таким образом, каждый фильтр $ W[c_{\text{out}}, :, :, :] $ имеет размерность $ C_{\text{in}} \times kH \times kW $ и будет свёртываться со всеми входными каналами для получения одного выходного канала.

### 1.3. Смещение B (Bias)

К каждому выходному каналу добавляется обучаемое смещение (bias). Bias представляет собой вектор длины $ C_{\text{out}} $:

```math
B \in \mathbb{R}^{C_{\text{out}}}
```

В некоторых реализациях bias может отсутствовать или быть тензором другой формы (например, если используются группы), но в стандартном случае это одномерный вектор.

### 1.4. Выходные данные Y (Output tensor)

Результат свёртки — тензор признаков, полученный после применения всех фильтров и добавления смещений. Его размерность:

```math
Y \in \mathbb{R}^{N \times C_{\text{out}} \times H_{\text{out}} \times W_{\text{out}}}
```

Значения $ H_{\text{out}} $ и $ W_{\text{out}} $ зависят от параметров свёртки: размера ядра ($ kH, kW $), шага (stride), дополнения (padding) и, возможно, расширения (dilation).

### 1.5. Параметры свёртки

Для управления пространственными размерами выхода и областью применения фильтров используются следующие гиперпараметры:

- **Stride** ($ s_h, s_w $): шаг, с которым фильтр перемещается по входному тензору. Обычно задаётся одинаковым для высоты и ширины, но может различаться.
- **Padding** ($ p_h, p_w $): количество нулевых строк/столбцов, добавляемых к границам входного тензора по высоте и ширине. Позволяет контролировать пространственный размер выхода и сохранять информацию на краях.
- **Dilation** ($ d_h, d_w $): параметр расширения ядра. Определяет расстояние между элементами ядра при свёртке. Если dilation > 1, ядро как бы «растягивается» (между соседними элементами вставляются пропуски). В стандартном случае dilation = 1.
- **Groups** (необязательно): позволяет разделить входные и выходные каналы на группы, выполняя свёртку независимо внутри каждой группы. В базовом описании группы не рассматриваются (groups = 1).

### 1.6. Формулы для пространственных размеров выхода

Пусть:
- $ H_{\text{in}}, W_{\text{in}} $ — размеры входа;
- $ kH, kW $ — размеры ядра;
- $ s_h, s_w $ — шаги;
- $ p_h, p_w $ — padding (количество нулей, добавляемых к каждой стороне);
- $ d_h, d_w $ — dilation.

Тогда высота выхода $ H_{\text{out}} $ и ширина выхода $ W_{\text{out}} $ вычисляются по общей формуле:

```math
H_{\text{out}} = \left\lfloor \frac{H_{\text{in}} + 2p_h - d_h \cdot (kH - 1) - 1}{s_h} \right\rfloor + 1
```
```math
W_{\text{out}} = \left\lfloor \frac{W_{\text{in}} + 2p_w - d_w \cdot (kW - 1) - 1}{s_w} \right\rfloor + 1
```

Для наиболее распространённого случая (dilation = 1) формула упрощается:

```math
H_{\text{out}} = \left\lfloor \frac{H_{\text{in}} + 2p_h - kH}{s_h} \right\rfloor + 1
```
```math
W_{\text{out}} = \left\lfloor \frac{W_{\text{in}} + 2p_w - kW}{s_w} \right\rfloor + 1
```

Если padding выбран так, что $ 2p = (k-1) $ (при stride = 1), то $ H_{\text{out}} = H_{\text{in}} $ (так называемое «same» padding).

### 1.7. Алгоритм вычисления свёртки

Для каждого элемента выходного тензора $ Y[n, c_{\text{out}}, i, j] $ выполняется следующая процедура:

1. Определяются пространственные координаты в исходном входном тензоре $ X $, которые соответствуют текущему положению фильтра. С учётом шага и padding:
   ```math
   h_{\text{start}} = i \cdot s_h - p_h, \quad h_{\text{end}} = h_{\text{start}} + d_h \cdot (kH - 1) + 1
   ```
   ```math
   w_{\text{start}} = j \cdot s_w - p_w, \quad w_{\text{end}} = w_{\text{start}} + d_w \cdot (kW - 1) + 1
   ```
   Однако при реализации удобнее итерировать по элементам ядра и вычислять соответствующие координаты входа.

2. Вычисляется свёртка (точнее, корреляция, так как ядро не переворачивается, что является стандартом в глубоком обучении) как сумма произведений весов фильтра на соответствующие элементы входа по всем входным каналам и пространственным позициям ядра:

```math
Y[n, c_{\text{out}}, i, j] = B[c_{\text{out}}] + \sum_{c_{\text{in}}=0}^{C_{\text{in}}-1} \sum_{u=0}^{kH-1} \sum_{v=0}^{kW-1} X\left[n, c_{\text{in}}, i\cdot s_h + u\cdot d_h - p_h,\; j\cdot s_w + v\cdot d_w - p_w\right] \cdot W[c_{\text{out}}, c_{\text{in}}, u, v]
```

Здесь:
- $ n $ — индекс примера в батче;
- $ c_{\text{out}} $ — индекс выходного канала (фильтра);
- $ i, j $ — пространственные координаты в выходном тензоре;
- $ c_{\text{in}} $ — индекс входного канала;
- $ u, v $ — смещения внутри ядра (пространственные индексы фильтра);
- $ s_h, s_w $ — шаги;
- $ d_h, d_w $ — dilation;
- $ p_h, p_w $ — padding.

Важно: при вычислении необходимо проверять, что координаты входа
\[
h = i\cdot s_h + u\cdot d_h - p_h, \quad w = j\cdot s_w + v\cdot d_w - p_w
\]
находятся в пределах допустимого диапазона $ [0, H_{\text{in}}-1] \times [0, W_{\text{in}}-1] $. Если они выходят за границы, то соответствующий элемент входа считается равным нулю (это соответствует zero-padding). Если используется другой тип padding (например, reflection), то применяется соответствующее правило дополнения границ.

### 1.8. Пример

Рассмотрим простейший случай без dilation ($ d_h = d_w = 1 $) и с padding = 0, stride = 1. Тогда:

```math
H_{\text{out}} = H_{\text{in}} - kH + 1,\quad W_{\text{out}} = W_{\text{in}} - kW + 1
```

Формула свёртки упрощается:

```math
Y[n, c_{\text{out}}, i, j] = B[c_{\text{out}}] + \sum_{c_{\text{in}}=0}^{C_{\text{in}}-1} \sum_{u=0}^{kH-1} \sum_{v=0}^{kW-1} X[n, c_{\text{in}}, i+u, j+v] \cdot W[c_{\text{out}}, c_{\text{in}}, u, v]
```

Пусть, например:
- $ N = 2 $ (два изображения в батче);
- $ C_{\text{in}} = 3 $ (RGB);
- $ H_{\text{in}} = 5, W_{\text{in}} = 5 $;
- $ C_{\text{out}} = 2 $ (два фильтра);
- $ kH = 3, kW = 3 $.

Тогда $ H_{\text{out}} = W_{\text{out}} = 5 - 3 + 1 = 3 $. Выходной тензор $ Y $ будет размером $ 2 \times 2 \times 3 \times 3 $.

Для вычисления $ Y[0, 0, 1, 2] $ (первый пример, первый выходной канал, позиция (1,2) в выходе) нужно:
- Взять фильтр №0 $ W[0,:,:,:] $ размером $ 3 \times 3 \times 3 $.
- Для каждого входного канала $ c_{\text{in}} = 0,1,2 $ и каждой позиции ядра $ u=0..2, v=0..2 $ извлечь элемент из X: $ X[0, c_{\text{in}}, 1+u, 2+v] $ и умножить на соответствующий вес $ W[0, c_{\text{in}}, u, v] $. Все произведения сложить и добавить $ B[0] $.

### 1.9. Варианты реализации

На практике свёртка редко реализуется через вложенные циклы напрямую из-за низкой эффективности. Вместо этого применяются оптимизированные методы:

- **im2col + GEMM**: входной тензор преобразуется в матрицу, где каждая строка соответствует одному положению фильтра (развёрнутые локальные блоки), а ядра свёртки также разворачиваются в матрицу. Затем выполняется матричное умножение, результат собирается в выходной тензор.
- **Прямое вычисление с использованием библиотек** (например, cuDNN, MKL-DNN), которые учитывают архитектуру процессора/GPU и особенности кэширования.
- **Преобразования в частотную область** (FFT) для больших ядер.

### 1.10. Дополнительные соображения

- **Groups**: если groups > 1, то входные и выходные каналы разбиваются на группы. Каждый фильтр взаимодействует только с каналами своей группы. Размерности весов изменяются: $ W \in \mathbb{R}^{C_{\text{out}} \times (C_{\text{in}}/G) \times kH \times kW} $, где $ G $ — число групп, причём $ C_{\text{out}} $ и $ C_{\text{in}} $ должны делиться на $ G $.
- **Depthwise convolution**: частный случай groups = $ C_{\text{in}} $, $ C_{\text{out}} = C_{\text{in}} $ (или кратно). Каждый канал обрабатывается отдельным фильтром.
- **Транспонированная (разреженная) свёртка**: используется для повышения пространственного разрешения, имеет свои формулы.




## 2. Gemm (General Matrix Multiply)

### 2.1. Назначение и общее описание

Gemm (General Matrix Multiply) — это обобщённая операция матричного умножения, которая является основным строительным блоком для многих вычислений в нейронных сетях, включая полносвязные (fully-connected) слои, рекуррентные слои (RNN, LSTM, GRU) и, после специального преобразования, свёрточные слои [citation:1][citation:5]. Операция выполняет умножение двух матриц с возможным масштабированием и добавлением третьей матрицы (или смещения).

### 2.2. Математическая формулировка

Стандартное определение операции Gemm следует соглашению, принятому в библиотеках линейной алгебры (таких как BLAS) [citation:1]:

```math
\mathbf{Y} = \alpha \cdot \mathbf{A} \mathbf{B} + \beta \cdot \mathbf{C}
```

где:
- $\alpha$ и $\beta$ — скалярные коэффициенты (обычно $\alpha = 1$, $\beta = 0$ или 1);
- $\mathbf{A}$, $\mathbf{B}$, $\mathbf{C}$ — матрицы.

В контексте нейронных сетей наиболее часто используется частный случай с $\alpha = 1$ и $\beta = 0$, что соответствует прямому матричному умножению. Также распространён вариант с добавлением вектора смещения (bias), который можно рассматривать как частный случай матрицы $\mathbf{C}$ с $\beta = 0$ и последующим сложением с вектором, транслируемым по строкам.

### 2.3. Размерности входных и выходных данных

Для операции Gemm используются следующие обозначения размерностей [citation:1][citation:2][citation:9]:
- Матрица $\mathbf{A}$ имеет размерность $M \times K$ (M строк и K столбцов);
- Матрица $\mathbf{B}$ имеет размерность $K \times N$ (K строк и N столбцов);
- Выходная матрица $\mathbf{C}$ (результирующая) имеет размерность $M \times N$ (M строк и N столбцов).

```math
\mathbf{A} \in \mathbb{R}^{M \times K}, \quad
\mathbf{B} \in \mathbb{R}^{K \times N}, \quad
\mathbf{C} \in \mathbb{R}^{M \times N}
```

В терминах нейронных сетей, где полносвязный слой часто представляется как $ \mathbf{Y} = \mathbf{X} \mathbf{W}^T + \mathbf{b} $, соответствие размерностей следующее [citation:3]:
- $\mathbf{X}$ (вход) размера $B \times N$, где $B$ — размер батча (batch size), $N$ — количество входных признаков (input features);
- $\mathbf{W}$ (веса) размера $M \times N$, где $M$ — количество выходных нейронов (output features);
- $\mathbf{b}$ (смещение) — вектор длины $M$;
- $\mathbf{Y}$ (выход) размера $B \times M$.

В этом случае операция Gemm выполняется как $\mathbf{Y} = \mathbf{X} \mathbf{W}^T$ (т.е. $\mathbf{A} = \mathbf{X}$, $\mathbf{B} = \mathbf{W}^T$), что соответствует размерностям $M = B$, $K = N$, $N = M$ в терминах Gemm.

### 2.4. Вычислительная сложность

Для вычисления произведения матриц $\mathbf{A} \mathbf{B}$ размером $M \times K$ и $K \times N$ необходимо выполнить $M \cdot N \cdot K$ операций умножения и сложения с плавающей точкой. Каждая такая операция "умножение-сложение" (multiply-add) часто реализуется одной инструкцией FMA (Fused Multiply-Add) и считается за два FLOPs (Floating Point Operations). Таким образом, общее количество операций составляет [citation:1][citation:8]:

```math
\text{FLOPs} = 2 \cdot M \cdot N \cdot K
```

### 2.5. Алгоритм выполнения

На высоком уровне алгоритм Gemm можно представить следующим образом: каждый элемент $c_{ij}$ результирующей матрицы $\mathbf{C}$ вычисляется как скалярное произведение $i$-й строки матрицы $\mathbf{A}$ и $j$-го столбца матрицы $\mathbf{B}$ [citation:4][citation:8].

Базовый алгоритм (наивная реализация) на псевдокоде:
```
Вход: матрица A размера M×K, матрица B размера K×N, скаляры α, β, матрица C размера M×N
Выход: матрица C = α·A·B + β·C

Для i от 0 до M-1:
Для j от 0 до N-1:
sum = 0
Для k от 0 до K-1:
sum += A[i][k] * B[k][j]
C[i][j] = α * sum + β * C[i][j]
```

На практике, особенно на GPU, используются оптимизированные реализации, основанные на разбиении матриц на блоки (tiling) для эффективного использования кэш-памяти и параллельных вычислительных блоков [citation:1][citation:2][citation:7]. Современные реализации (например, в библиотеках cuBLAS) достигают производительности, близкой к пиковой, за счёт тщательного управления иерархией памяти (регистры, разделяемая память, глобальная память) [citation:7][citation:8].

## 3. Операция MatMul (Matrix Multiplication)

### 3.1. Назначение и общее описание

MatMul (Matrix Multiplication) — это термин, часто используемый как синоним Gemm, но в более узком смысле он обычно обозначает простое матричное умножение без масштабирующих коэффициентов и дополнительного слагаемого [citation:4]. MatMul является частным случаем Gemm, где $\alpha = 1$ и $\beta = 0$.

В фреймворках глубокого обучения (TensorFlow, PyTorch) под `matmul` обычно понимается операция умножения двух тензоров с поддержкой пакетной обработки (batching) и трансляции (broadcasting) размерностей.

### 3.2. Математическая формулировка

```math
\mathbf{C} = \mathbf{A} \mathbf{B}
```

### 3.3. Размерности входных и выходных данных

Размерности для операции MatMul следуют тем же правилам, что и для матричного умножения. Для двумерных тензоров (матриц):
- $\mathbf{A}$ размера $M \times K$;
- $\mathbf{B}$ размера $K \times N$;
- $\mathbf{C}$ размера $M \times N$.

Для многомерных тензоров (пакетная обработка):
- Если $\mathbf{A}$ имеет форму $[b_1, b_2, ..., b_n, M, K]$, а $\mathbf{B}$ имеет форму $[b_1, b_2, ..., b_n, K, N]$, то выход $\mathbf{C}$ будет иметь форму $[b_1, b_2, ..., b_n, M, N]$ (попарное умножение матриц для каждого элемента пакета).
- Также поддерживается трансляция: если размерности пакетов не совпадают, меньший тензор "транслируется" (расширяется) для соответствия размерности большего тензора.

### 3.4. Алгоритм выполнения

Базовый алгоритм MatMul идентичен алгоритму Gemm. Однако, из-за отсутствия коэффициентов и дополнительного слагаемого, вычисления могут быть несколько упрощены. В остальном применяются те же оптимизированные методы, включая блочное разбиение и использование SIMD/векторных инструкций [citation:7][citation:8].

## 4. Операция Mul (поэлементное умножение)

### 4.1. Назначение и общее описание

Операция Mul (Element-wise Multiplication) выполняет поэлементное умножение двух тензоров одинаковой формы или с поддержкой трансляции (broadcasting). Это фундаментальная операция, используемая во многих архитектурах нейронных сетей, например, в механизмах внимания (attention), нормализации (normalization) и остаточных связях.

### 4.2. Математическая формулировка

```math
\mathbf{Y} = \mathbf{X} \odot \mathbf{W}
```

где $\odot$ обозначает поэлементное умножение (произведение Адамара, Hadamard product).

### 4.3. Размерности входных и выходных данных

Пусть $\mathbf{X}$ имеет размерность $d_1 \times d_2 \times ... \times d_n$, а $\mathbf{W}$ имеет размерность, совместимую для поэлементного умножения с $\mathbf{X}$ (т.е. либо точно такую же, либо форму, которую можно транслировать до размерности $\mathbf{X}$). Тогда выходной тензор $\mathbf{Y}$ будет иметь ту же размерность, что и $\mathbf{X}$ (после применения трансляции).

В простейшем случае:
```math
\mathbf{X} \in \mathbb{R}^{d_1 \times d_2 \times ... \times d_n}, \quad
\mathbf{W} \in \mathbb{R}^{d_1 \times d_2 \times ... \times d_n}, \quad
\mathbf{Y} \in \mathbb{R}^{d_1 \times d_2 \times ... \times d_n}
```

### 4.4. Алгоритм выполнения

Алгоритм заключается в итерации по всем элементам входного тензора и умножении каждого элемента $x_i$ на соответствующий элемент $w_i$:

```
Вход: тензор X, тензор W (совместимой формы)
Выход: тензор Y = X ⊙ W

Для каждого индекса i в диапазоне размерности X:
Y[i] = X[i] * W[i] // где W[i] получен по правилам трансляции
```


На практике эта операция хорошо векторизуется и выполняется параллельно для всех элементов, что обеспечивает высокую производительность на GPU.

## 5. Операция Add (поэлементное сложение)

### 5.1. Назначение и общее описание

Операция Add (Element-wise Addition) выполняет поэлементное сложение двух или более тензоров. Как и Mul, она поддерживает трансляцию и является ключевым компонентом остаточных связей (residual connections), нормализации и суммирования признаков из разных ветвей сети.

### 5.2. Математическая формулировка

Для двух тензоров:
```math
\mathbf{Y} = \mathbf{X} + \mathbf{W}
```

Для случая с добавлением смещения (bias), который часто встречается в нейронных сетях:
```math
\mathbf{Y} = \mathbf{X} + \mathbf{b}
```
где $\mathbf{b}$ — вектор, который транслируется на всю пространственную размерность $\mathbf{X}$.

### 5.3. Размерности входных и выходных данных

Как и для Mul, входные тензоры должны иметь совместимые для трансляции формы. Выходной тензор $\mathbf{Y}$ имеет ту же размерность, что и входной тензор $\mathbf{X}$ после применения трансляции.

Распространённый пример (добавление смещения к результату матричного умножения):
```math
\mathbf{X} \in \mathbb{R}^{B \times M}, \quad
\mathbf{b} \in \mathbb{R}^{M}, \quad
\mathbf{Y} \in \mathbb{R}^{B \times M}
```
Здесь вектор $\mathbf{b}$ транслируется (копируется) на все B элементов батча.

### 5.4. Алгоритм выполнения

Алгоритм аналогичен операции умножения:

```
Вход: тензор X, тензор W (совместимой формы)
Выход: тензор Y = X + W

Для каждого индекса i в диапазоне размерности X:
Y[i] = X[i] + W[i] // где W[i] получен по правилам трансляции
```


## 6. Операция ReLU (Rectified Linear Unit)

### 6.1. Назначение и общее описание

ReLU (Rectified Linear Unit) — это наиболее распространённая функция активации в глубоких нейронных сетях. Она вносит нелинейность в модель, оставляя положительные значения без изменений и обнуляя отрицательные [citation:3]. Простота вычисления ReLU способствует эффективному обучению и обратному распространению ошибки.

### 6.2. Математическая формулировка

Функция активации ReLU определяется кусочно [citation:3]:

```math
f(x) = \max(0, x) = \begin{cases} 
x & \text{если } x > 0 \\
0 & \text{если } x \leq 0 
\end{cases}
```

В матричном виде для входного тензора $\mathbf{X}$:
```math
\mathbf{Y} = \text{ReLU}(\mathbf{X}) = \max(0, \mathbf{X})
```

где максимум берётся поэлементно.

### 6.3. Размерности входных и выходных данных

Операция ReLU применяется поэлементно, поэтому размерность выходного тензора полностью совпадает с размерностью входного:
```math
\mathbf{X} \in \mathbb{R}^{d_1 \times d_2 \times ... \times d_n}, \quad
\mathbf{Y} \in \mathbb{R}^{d_1 \times d_2 \times ... \times d_n}
```

### 6.4. Алгоритм выполнения

Алгоритм ReLU предельно прост и эффективен:

```
Вход: тензор X произвольной размерности
Выход: тензор Y = ReLU(X)

Для каждого элемента x в тензоре X:
Если x > 0:
Y[соответствующий индекс] = x
Иначе:
Y[соответствующий индекс] = 0
```

Благодаря своей простоте, ReLU может быть вычислена за один проход по данным с тривиальной логикой ветвления. На GPU эта операция выполняется чрезвычайно быстро и часто объединяется (fused) с предшествующей операцией (например, Gemm) для уменьшения количества обращений к памяти и повышения общей производительности [citation:3].
